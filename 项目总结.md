# AI新闻聚合项目 - 零基础学习指南

## 🎯 项目简介

### 这是什么项目？
这是一个**AI新闻聚合工具**，它就像一个"智能新闻管家"，每天自动从多个网站抓取最新的人工智能（AI）相关新闻，整理成一份漂亮的日报，可以发布到微信公众号、网页或保存为Markdown文档。

### 它能做什么？
- 📰 **自动抓取新闻**：从TechCrunch、The Verge、Hacker News、36氪、虎嗅等5个网站抓取AI新闻
- 🤖 **智能摘要**：使用AI（DeepSeek）为每篇文章生成一句话中文摘要
- 📱 **多种格式输出**：支持Markdown、HTML网页、微信公众号文章格式
- ⏰ **自动更新**：可以设置每天自动运行，保持内容最新
- 🌐 **公众号集成**：生成的HTML可以直接部署到GitHub Pages，然后链接到微信公众号菜单

### 为什么需要这个项目？
- 对于**AI爱好者**：每天快速了解AI领域最新动态
- 对于**内容创作者**：自动生成素材，节省收集时间
- 对于**学习者**：通过实际项目学习TypeScript、Node.js、API调用等技能
- 对于**创业者**：可以作为内容产品的基础框架

### 架构亮点：Agent 协作生态

本项目最大的改进是从"单一脚本"重构为 **Agent 协作生态**：

```
┌──────────────────────────────────────────────────────┐
│                 Orchestrator Agent                    │
│  编排调度 · 状态管理 · 异常处理 · 重试机制             │
└─────┬───────────┬───────────┬────────────────────────┘
      │ step 1    │ step 2    │ step 3
      ▼           ▼           ▼
 ┌─────────┐ ┌─────────┐ ┌─────────┐
 │  Fetch  │→│Summarize│→│ Render  │
 │  Agent  │ │  Agent  │ │  Agent  │
 └─────────┘ └─────────┘ └─────────┘
      │           │           │
      ▼           ▼           ▼
 ┌──────────────────────────────────────────────────────┐
 │                  Pipeline State                       │
 │  .pipeline/state.json（共享状态，Agent 通信桥梁）      │
 └──────────────────────────────────────────────────────┘
```

每个 Agent 遵循 **Perceive（感知）→ Think（思考）→ Act（行动）** 模型：

| 阶段 | 做什么 |
|---|---|
| **Perceive** | 从 `.pipeline/` 读取状态和输入数据 |
| **Think** | 判断是否该执行、用哪个策略、是否重试 |
| **Act** | 调用函数库干活，写入结果到 `.pipeline/` |

Agent 之间**不直接调用**，通过状态文件通信，互不耦合，方便独立调试和扩展。

### 各模块的作用

#### 1. Agent 层（`src/agents/`）
- **Orchestrator Agent**：编排整条管线，管理状态流转，处理重试和降级
- **Fetch Agent**：调度 RSS 抓取、数据处理（过滤/去重/排序）、写入原始文章
- **Summarize Agent**：调用 LLM 生成摘要，质量评估兜底
- **Render Agent**：读取文章（优先带摘要的），生成 Markdown/HTML

#### 2. 工具函数层
- **`src/fetchers/`**：从各个网站抓取新闻
- **`src/processors/`**：过滤、去重、AI关键词筛选、排序
- **`src/generators/`**：生成 Markdown/HTML 输出
- **`src/services/`**：AI 摘要服务

#### 3. 管线状态层（`.pipeline/`）
Agent 之间的通信桥梁，Orchestrator 统一管理读写权限。

#### 4. 配置文件（`src/config.ts`）
```typescript
// 这里配置了所有的新闻源
const RSS_SOURCES = [
  {
    name: 'TechCrunch AI',
    url: 'https://techcrunch.com/category/artificial-intelligence/feed/',
    type: 'techcrunch'
  },
  // ... 其他新闻源
];
```

## 🔧 核心功能详解

### 1. 新闻抓取流程
```
访问RSS源 → 解析XML格式 → 提取文章信息 → 清理数据 → 返回结果
```

**示例**：抓取TechCrunch AI新闻
- 访问：`https://techcrunch.com/category/artificial-intelligence/feed/`
- 解析：获取标题、链接、发布时间、摘要
- 处理：清理多余的HTML标签，格式化时间

### 2. AI智能摘要
如果需要使用AI摘要功能，需要配置API密钥：

```bash
# 1. 复制配置文件
cp .env.example .env

# 2. 编辑.env文件，添加你的DeepSeek API密钥
ANTHROPIC_API_KEY=sk-your-deepseek-api-key-here
AI_SUMMARY_ENABLED=true
```

**DeepSeek API是什么？**
- DeepSeek是一家中国的AI公司，提供类似ChatGPT的服务
- 需要注册账号获取API密钥（有免费额度）
- 项目会调用API为每篇文章生成一句话摘要

### 3. 命令行使用

#### 推荐方式：通过 Agent 运行管线
```bash
# 完整管线（抓取 → 摘要 → 渲染）
npx tsx src/cli.ts agent orchestrator --force

# 单独调试某个环节
npx tsx src/cli.ts agent fetch-agent
npx tsx src/cli.ts agent summarize-agent
npx tsx src/cli.ts agent render-agent
```

#### 单步命令（保留原有用法）
```bash
# 基本使用
npm run dev fetch

# 启用AI摘要
npm run dev fetch -- --ai-summary

# 生成HTML网页版本
npm run dev fetch -- --html-output

# 组合使用：AI摘要 + HTML网页
npm run dev fetch -- --ai-summary --html-output

# 自定义时间范围（抓取最近48小时）
npm run dev fetch -- --hours 48
```

## 🌐 部署到微信公众号

### 方案一：公众号菜单跳转（推荐）
1. **生成HTML日报**
   ```bash
   npm run dev fetch -- --html-output
   ```

2. **部署到GitHub Pages**
   - 创建GitHub仓库
   - 将生成的HTML文件推送到仓库
   - 在仓库设置中启用GitHub Pages
   - 获得访问地址：`https://用户名.github.io/仓库名/`

3. **配置公众号菜单**
   - 登录微信公众平台
   - 添加菜单项，类型选择"跳转网页"
   - 粘贴GitHub Pages地址
   - 提交审核（1-3个工作日）

### 方案二：公众号文章发布
1. **生成公众号文章格式**
   ```bash
   npm run dev fetch -- --ai-summary --wechat-article
   ```

2. **手动发布**
   - 登录公众号后台 → 新建图文消息
   - 复制生成的文章内容
   - 添加封面图、摘要
   - 发布或保存为草稿

**两种方案对比**：
| 方案 | 优点 | 缺点 |
|------|------|------|
| 菜单跳转 | 完全自动，每天更新 | 需要审核，可能有外链限制 |
| 文章发布 | 完全符合微信规定 | 需要每天手动发布 |

## 🧩 扩展开发（如何修改项目）

### 1. 添加新的新闻源
编辑 `src/config.ts` 文件：

```typescript
// 在RSS_SOURCES数组中添加新的源
export const RSS_SOURCES = [
  // ... 原有源
  {
    name: '新源名称',      // 显示名称
    url: 'RSS地址',       // RSS订阅地址
    type: '自定义类型'     // 英文标识
  }
];
```

### 2. 修改AI关键词
如果你想让程序抓取其他类型的新闻，比如"区块链"或"元宇宙"：

编辑 `src/processors/filter.ts`：
```typescript
// 在aiKeywords数组中添加关键词
const aiKeywords = [
  // ... 原有关键词
  '区块链', '比特币', '以太坊',      // 添加区块链相关
  '元宇宙', 'VR', 'AR', '虚拟现实'   // 添加元宇宙相关
];
```

### 3. 添加新的 Agent
如需扩展管线功能（如推送飞书、生成语音版）：

1. 在 `.specs/` 创建 Agent 规范文件
2. 在 `src/agents/` 实现 Perceive → Think → Act 逻辑
3. 在 `src/cli.ts` 注册新的 `agent` 命令
4. 在 `CLAUDE.md` 注册到 Agent 注册表
5. 在 Orchestrator 中编排新步骤

### 4. 修改输出格式
- **Markdown格式**：编辑 `src/generators/markdown.ts`
- **HTML样式**：编辑 `src/generators/html.ts`
- **公众号格式**：编辑 `src/generators/wechat.ts`

## 🐛 常见问题解答

### Q1: 运行时报错"找不到模块"
**A**: 需要先运行 `npm install` 安装依赖包。

### Q2: AI摘要功能无法使用
**A**:
1. 检查是否创建了 `.env` 文件
2. 检查是否设置了正确的API密钥
3. 检查网络是否能访问DeepSeek API

### Q3: 抓取的新闻很少或没有
**A**:
1. 检查网络连接
2. 检查RSS源是否仍然有效
3. 尝试增加时间范围：`--hours 48`

### Q4: 部署到GitHub Pages后无法访问
**A**:
1. 检查仓库设置中是否启用了GitHub Pages
2. 等待几分钟让GitHub部署完成
3. 检查URL是否正确

### Q5: 公众号菜单审核不通过
**A**:
1. 尝试使用"公众号文章"方案
2. 确保链接内容是合规的
3. 联系微信客服咨询具体原因
